# VN Workbench

一个偏 Galgame / 视觉小说工作流的本地前端工具。

它把文本创作、剧情分析、绘图提示词整理、图片收纳、模拟演出、项目保存、HTML 导出、背景音乐与交互音整合到一个页面里，适合做快速原型、展示项目和本地演示。

## 主要功能

- 文本端创作、导入、分析、分段
- 场景拆解为角色 / 身份 / 表情 / 动作 / 背景等信息
- 自动整理绘图提示词，并送入绘画端
- 已生成图片收纳、描述、检索、下载、复制 URL、删除
- 模拟端直接演出，支持全屏、Ending、存档 / 读档
- 背景音乐音频库，支持本地上传和本地保存
- 交互音配置，支持点击对话框 / 翻页时播放
- 项目本地保存与恢复
- 导出可直接运行的单文件 HTML 展示版
- 导出 Ren'Py 脚本

## 项目结构

- `index.html`：主程序，绝大多数 UI 与逻辑都在这里
- `local_tts_proxy.py`：本地语音代理，提供 OpenAI 风格的语音接口
- `start_local_tts_proxy.bat`：快速启动本地语音代理
- `maskgct_quick_infer.py`：为后续本地 TTS 模型接入准备的推理脚本
- `HANDOFF.md`：项目交接说明
- `GEMINI_UI_HANDOFF.md`：UI 修改约束说明

## 本地运行

推荐直接在当前目录启动一个静态服务。

### Python

```bash
python -m http.server 8000
```

然后浏览器打开：

```text
http://127.0.0.1:8000/
```

## 使用流程

### 1. 文本创作

- 在文本端输入故事、导入纯文本，或让 AI 生成故事
- 点击分析，生成结构化场景信息

### 2. 绘画端

- 根据分析结果整理正向 / 负向提示词
- 生成图片，或上传自有图片进入图库
- 给图片写描述，方便后续检索和复用

### 3. 模拟端

- 直接演出剧情
- 支持背景音乐、交互音、全屏和 Ending
- 最后一页不会循环返回开头

### 4. 保存与导出

- 在项目管理中保存当前项目
- 可以导出 JSON
- 也可以导出单文件 HTML，别人下载后本地即可直接打开体验

## 音频说明

### 背景音乐

- 支持上传本地音频文件
- 也支持填写 URL / 本地路径
- 推荐直接上传本地文件，稳定性更高

### 交互音

- 支持上传本地音频文件
- 点击对话框或切换句子时触发
- 推荐使用很短的 click / tap 音效

## 保存机制

项目、图片收纳、背景音乐和交互音会优先保存到本地持久化存储。

但仍建议：

- 重要内容及时手动保存项目
- 展示前先导出一份 HTML 备份
- API Key 不要直接写进可公开分享的文件里

## 导出 HTML

导出的 HTML 目标是做“可直接展示”的单文件版本，尽量把以下内容一起带走：

- 剧情文字
- 图片
- 背景音乐
- 交互音

如果某些资源当时浏览器无法读取，导出文件会尽量保留原地址。

## 注意事项

- 这是一个本地优先、API 驱动的前端工具，不是完全离线生产管线
- 部分浏览器会限制自动播放音频，第一次播放可能需要用户点击
- 手机展示建议横屏使用
